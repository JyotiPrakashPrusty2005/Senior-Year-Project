2. Detailed Request Flow

sequenceDiagram
    actor P as Patient
    participant FE as Frontend
    participant API as FastAPI
    participant IC as Intent Classifier
    participant ML as ML Models
    participant LSTM as LSTM Model
    participant AI as Gemini/GPT
    participant DB as Database

    Note over P,DB: FLOW 1 - DISEASE PREDICTION
    P->>FE: Selects Disease Prediction
    FE->>P: Shows health form
    P->>FE: Fills form + optionally uploads X-ray
    FE->>API: POST /predict/disease
    API->>IC: Classify intent from input fields
    
    alt Has X-ray image
        IC->>ML: Route to Pneumonia Model
    else Has BP, cholesterol, glucose
        IC->>ML: Route to Cardiovascular Model
    else Has 21 BRFSS indicators
        IC->>ML: Route to Diabetes Model
    end
    
    ML-->>API: prediction, probability, risk_level
    API-->>FE: JSON response
    FE-->>P: Risk Report with advice

    Note over P,DB: FLOW 2 - POST-SURGERY RECOVERY
    P->>FE: Selects Recovery Assistant
    FE->>P: Shows profile form - one time
    P->>FE: Fills profile
    FE->>API: POST /patient/register
    API->>DB: Store patient profile
    
    loop Every Day - Day 1 to 30
        P->>FE: Enters daily vitals
        FE->>API: POST /patient/vitals
        API->>DB: Store vitals
        
        alt Has 7+ days of data
            API->>DB: Fetch last 7 days
            DB-->>API: vitals_history
            API->>LSTM: predict with 7-day window
            LSTM-->>API: forecast + complication risk
            API->>AI: Generate advice
            AI-->>API: Personalized recovery plan
            API-->>FE: Full recovery report
            FE-->>P: Daily recovery plan + alerts
        else Less than 7 days
            API-->>FE: Generic guidelines only
            FE-->>P: Basic recovery tips
        end
    end

3. Intent Classifier Logic

flowchart LR
    INPUT[Form Input] --> CHECK{What fields are present?}
    CHECK -->|X-ray image uploaded| PNEUMONIA[Pneumonia Model
CNN - ResNet/EfficientNet]
    CHECK -->|ap_hi, ap_lo, cholesterol,
glucose, height, weight| CARDIO[Cardiovascular Model
XGBoost/LightGBM]
    CHECK -->|HighBP, HighChol, BMI,
21 BRFSS indicators| DIABETES[Diabetes Model
XGBoost/LightGBM]
    CHECK -->|No match| ERROR[Error: Insufficient data]

    PNEUMONIA --> OUT[Standardized JSON Response]
    CARDIO --> OUT
    DIABETES --> OUT
